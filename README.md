# DevTools Collection

> A modern, privacy-first collection of developer tools built with Next.js App Router, React, shadcn/ui, and static export.

[![License](https://img.shields.io/badge/license-MIT-blue)](LICENSE)

## About

DevTools Collection is a browser-based toolkit for common developer workflows like encoding, formatting, generating values, previewing content, and inspecting data. The project is deployed as a static export, while each tool runs client-side in the browser.

## Current Status

- 34 tool routes are exported statically
- Tool pages are generated with `generateStaticParams()`
- Home page, sitemap, and robots are included in the static export
- Astro/Preact migration is complete enough for the Next.js app to build cleanly

## Features

- 100% browser-side tool execution for user interactions
- Static export friendly with `next build`
- Search, category filtering, favorites, recent tools, dark mode
- Command palette with keyboard navigation
- shadcn-style component layer on Tailwind CSS
- Responsive layout for desktop and mobile

## Tool Categories

- Converters
- Encoders & Decoders
- Generators
- Formatters & Validators
- Text Tools
- Utilities
- Security
- Networking
- Design

## Tech Stack

- Framework: [Next.js 16 App Router](https://nextjs.org/docs)
- UI: [React 19](https://react.dev/)
- Styling: [Tailwind CSS](https://tailwindcss.com/)
- Component patterns: [shadcn/ui](https://ui.shadcn.com/)
- Icons: [Lucide React](https://lucide.dev/)
- Static hosting target: Cloudflare Pages / static hosting via `out/`

## Getting Started

### Prerequisites

- Node.js 20+
- npm

### Install

```bash
git clone https://github.com/bberka/devtools.git
cd devtools
npm install
```

### Run Dev Server

```bash
npm run dev
```

The app will be available at `http://localhost:3000`.

### Production Build

```bash
npm run build
```

The static export is generated in the `out/` directory.

### Preview

```bash
npm run start
```

## Development Notes

- Routes are exported statically with `output: 'export'`
- Tool pages are known at build time from `src/lib/utils/tools-config.ts`
- Tool UIs remain interactive client components after hydration
- Dynamic imports are used for code-splitting tool bundles, not to disable static export

## Deployment

For static hosting:

- Build command: `npm run build`
- Output directory: `out`

For Wrangler-based static asset publishing, `wrangler.json` should also point to `./out`.
