# DevTools Collection

> A privacy-first collection of browser-based developer tools built with Next.js App Router, React, shadcn-style components, Tailwind CSS, and static export.

[![License](https://img.shields.io/badge/license-MIT-blue)](LICENSE)

## About

DevTools Collection is a static web app for everyday developer workflows: encoding, formatting, generating values, previewing content, inspecting data, and doing small browser-side utilities. Tool interactions run in the browser; the site itself is generated at build time.

## Current Status

- 34 tool pages are exported statically from `src/lib/utils/tools-config.ts`
- Tool routes use `generateStaticParams()` in the Next.js App Router
- Home page, per-tool metadata, sitemap, robots, not-found page, favorites, recent tools, settings, dark mode, and command palette are implemented
- The old Astro/Preact app has been migrated out of the active source tree
- `npm run build` succeeds and writes the static export to `out/`

## Features

- Client-side tool execution for user interactions
- Static export friendly with `next build`
- Search, category filtering, favorites, recent tools, compact mode, and dark mode
- Command palette with keyboard navigation
- shadcn-style component layer on Tailwind CSS
- Responsive layout for desktop and mobile

## Getting Started

### Prerequisites

- Node.js 20+
- npm

### Install

```bash
git clone https://github.com/bberka/devtools.git
cd devtools
npm install
```

### Run Dev Server

```bash
npm run dev
```

The app will be available at `http://localhost:3000`.

### Production Build

```bash
npm run build
```

The static export is generated in the `out/` directory.

## Development Notes

- Routes are exported statically with `output: 'export'`
- Tool pages are known at build time from `src/lib/utils/tools-config.ts`
- Tool UIs remain interactive client components after hydration
- Tool component wiring currently lives in `src/components/ToolComponentRenderer.tsx`
- Local browser data uses the `dev-toolbox:*` localStorage namespace

## Deployment

For static hosting:

- Build command: `npm run build`
- Output directory: `out`

For Wrangler-based static asset publishing, `wrangler.json` points to `./out`.
